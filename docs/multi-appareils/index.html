📱💻 Utiliser KuT sur plusieurs appareils

KuT s'utilise sur PC, tablette et smartphone en même temps, avec une base clients unique partagée. Le principe est simple :

1 script déployé une fois → 1 URL …/exec → vous collez cette même URL sur chaque appareil → tous lisent & écrivent dans la même base sur votre Google Drive.
La règle d'or : la même URL partout. Deux URL différentes = deux bases séparées qui ne se parlent pas.

Étape 0 — Une seule fois : déployer le script

À faire une seule fois (sur n'importe quel ordinateur). Vous obtenez une URL qui se termine par /exec : c'est la clé de toute la synchro.

Voir la procédure de déploiement →

Déjà déployé avant ? Refaites Déployer → Gérer les déploiements → Modifier (crayon) → Version : Nouvelle version → Déployer, puis réautorisez. C'est nécessaire pour activer la synchro par client (rapide, multi-appareils) et l'envoi des relances. L'URL /exec ne change pas.

👉 Notez / envoyez-vous cette URL (par email ou note) : vous allez devoir la recopier sur chaque appareil.

Étape 1 — Sur chaque appareil : ouvrir l'application

Ouvrez cette adresse dans le navigateur de l'appareil :

https://j-rbs91.github.io/ciseaux/
💻
PC / Mac — Chrome, Edge, Firefox ou Safari. Ajoutez la page aux favoris.
iPhone / iPad (Safari) — bouton Partager (carré avec flèche ↑) → Sur l'écran d'accueil. Vous obtenez une icône comme une vraie appli.
Android (Chrome) — menu ⋮ en haut à droite → Ajouter à l'écran d'accueil (ou « Installer l'application »).

Étape 2 — Sur chaque appareil : coller la même URL

1
Ouvrir ⚙️ Mon salon.
2
Coller l'URL /exec (la même que sur les autres appareils) dans le champ URL de votre script Apps Script.
3
Cliquer Enregistrer.

Répétez l'étape 2 sur chaque appareil (et dans chaque navigateur que vous comptez utiliser).

Étape 3 — Vérifier que c'est synchronisé

Allez dans Clients : un indicateur affiche ✓ à jour quand tout est synchronisé, ou ⚠ N modif. en attente s'il reste des choses à envoyer.

Test simple : ajoutez un client sur le PC, puis ouvrez l'app sur le téléphone → il apparaît. Les appareils se mettent à jour automatiquement dès que vous revenez sur l'app (et toutes les ~30 secondes), sans avoir à recharger.

Les règles d'or au quotidien

  • Même URL partout — sinon les appareils ne partagent pas la même base.
  • Pas besoin d'attendre pour fermer — chaque action (ajout, modif, passage…) est enregistrée automatiquement, et l'app pousse aussi ce qui reste quand elle passe en arrière-plan. Fermez quand vous voulez.
  • Hors-ligne, rien n'est perdu — une bannière rouge s'affiche tant que des modifs ne sont pas envoyées. Vous pouvez quand même fermer : tout repart tout seul à la prochaine ouverture ou au retour du réseau (le navigateur affiche aussi un avertissement si vous fermez pile à ce moment-là).
  • Évitez de modifier le même client sur deux appareils à la même seconde — en cas de conflit, c'est le dernier enregistrement qui gagne (pas de fusion).
  • Ne videz pas « les données du site » / le cache du navigateur tant qu'il reste des modifs « en attente » : ce sont les seules données qui ne sont pas encore sur le Drive.

Dépannage

Je ne vois pas sur un appareil ce que j'ai saisi sur un autre.
Vérifiez que l'URL collée est identique partout. Revenez sur l'app (ou tirez la page vers le bas pour rafraîchir) et regardez l'indicateur de synchro.

L'indicateur affiche « ⚠ hors-ligne ».
Pas de réseau, ou l'URL est incorrecte. Recollez l'URL qui se termine bien par /exec.

Sur iPhone, rien ne se passe.
Utilisez Safari et ajoutez l'app à l'écran d'accueil (étape 1).

Compte Google Workspace (pro) : « Tout le monde » indisponible.
Créez un compte Gmail personnel dédié (ex. monsalon.sync@gmail.com) et déployez le script depuis ce compte.

Sécurité

L'URL /exec est longue et aléatoire : elle fait office de clé secrète. Ne la publiez pas. Toutes vos données restent sur votre Google Drive — KuT n'héberge rien.
